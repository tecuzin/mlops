# BMAD Branch Audit

Agis comme Tech Lead + QA Lead et applique BMAD (analyse -> planification -> architecture -> execution -> validation).

Objectif:
- comparer les branches locales/distantes actives
- identifier la branche de reference
- qualifier les risques de merge

Livrables obligatoires:
1. Resume executif (max 10 lignes)
2. Tableau comparatif des branches (commits uniques, fichiers, domaines, risque)
3. Matrice de chevauchement et conflits probables
4. Questions bloquantes avant merge

Contraintes:
- ne pas inventer d'informations
- ne pas lancer de commande destructive
- signaler explicitement les zones de conflit logique
